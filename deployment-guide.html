1 项目概览

BID AI Screener 是清华大学 BID 的 AI 招新筛选系统：

  • 候选人：在手机或电脑上完成面试问卷（基本信息 + AI 面试）
  • 管理员：登录后台查看所有候选人数据、打分、筛选

数据流

  1. 候选人提交：数据先写入 localStorage（确保不丢失），再异步写入 Supabase
  2. 管理员查看：优先从 Supabase 拉取，不可用时降级到 localStorage
  3. 认证：管理员通过 Supabase Auth 登录，不可用时降级到硬编码密码

3 Supabase 设置（数据库 + 认证）

3.1 创建 Supabase 项目

  1. 打开 supabase.com 并注册/登录（可用 GitHub 账号登录）
  2. 点击 "New Project"
  3. 填写：
    • Name：tsinghua-bid-screener
    • Database Password：设一个强密码并记下来
    • Region：选 Singapore (Southeast Asia)（国内最快）
  4. 点 "Create new project"，等待 1-2 分钟

3.2 建表（SQL Editor）

  1. 左侧菜单点 "SQL Editor"
  2. 点 "New query"
  3. 将以下 SQL 全部复制粘贴进去：
-- ============================================
-- BID AI Screener 数据库初始化
-- ============================================

-- 1. 建表：candidates（候选人信息表）
CREATE TABLE candidates (
  id              TEXT PRIMARY KEY,
  display_name    TEXT NOT NULL,
  status          TEXT NOT NULL
    CHECK (status IN ('pass', 'hold', 'reject')),
  status_badge    TEXT,
  name            TEXT NOT NULL,
  email           TEXT,
  phone           TEXT,
  wechat_id       TEXT,
  identity        TEXT,
  school_org      TEXT,
  score_overall   REAL,
  data            JSONB NOT NULL,
  created_at      TIMESTAMPTZ DEFAULT now(),
  updated_at      TIMESTAMPTZ DEFAULT now()
);

-- 2. 索引（加速查询）
CREATE INDEX idx_candidates_status ON candidates(status);
CREATE INDEX idx_candidates_created_at ON candidates(created_at DESC);

-- 3. 自动更新 updated_at 的触发器
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER candidates_updated_at
  BEFORE UPDATE ON candidates
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- 4. 开启行级安全策略
ALTER TABLE candidates ENABLE ROW LEVEL SECURITY;

-- 5. 安全策略
CREATE POLICY "anon_insert"  ON candidates FOR INSERT WITH CHECK (true);
CREATE POLICY "auth_select" ON candidates FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "auth_update" ON candidates FOR UPDATE USING (auth.role() = 'authenticated');
CREATE POLICY "auth_delete" ON candidates FOR DELETE USING (auth.role() = 'authenticated');

-- ============================================
-- 2. 建表：snapshots（系统备份快照表）
-- ============================================

CREATE TABLE snapshots (
  id          TEXT PRIMARY KEY,              -- 快照唯一标识
  label       TEXT NOT NULL,                 -- 备份标签
  timestamp   BIGINT NOT NULL,              -- 创建时间戳 (ms)
  data        JSONB NOT NULL,               -- 完整系统状态快照
  created_at  TIMESTAMPTZ DEFAULT now()     -- 数据库记录创建时间
);

CREATE INDEX idx_snapshots_timestamp ON snapshots(timestamp DESC);

ALTER TABLE snapshots ENABLE ROW LEVEL SECURITY;

CREATE POLICY "auth_select" ON snapshots FOR SELECT USING (auth.role() = 'authenticated');
CREATE POLICY "auth_insert" ON snapshots FOR INSERT WITH CHECK (auth.role() = 'authenticated');
CREATE POLICY "auth_update" ON snapshots FOR UPDATE USING (auth.role() = 'authenticated');
CREATE POLICY "auth_delete" ON snapshots FOR DELETE USING (auth.role() = 'authenticated');
  1. 点 "Run"（或 Ctrl+Enter / Cmd+Enter）
  2. 看到绿色 "Success. No rows returned" 即为成功

3.3 创建管理员账户

  1. 左侧菜单点 "Authentication"（锁形图标）
  2. 点上方 "Users" 标签
  3. 点 "Add user" → "Create new user"
  4. 填写：
    • Email：你的管理邮箱（如 admin@mails.tsinghua.edu.cn）
    • Password：管理密码
    • 勾选 "Auto Confirm User"
  5. 点 "Create user"
⚠️
如果保留了 Confirm email 开关，创建管理员时必须勾选 Auto Confirm，否则无法登录。

3.4 获取 API Key

  1. 左侧菜单最下方点 "Project Settings"（齿轮图标）
  2. 点左侧 "API"
  3. 记下两个值：
    • Project URL：如 https://xxxxxxxx.supabase.co
    • anon public key：一串长字符串
💡
这两个值后面在 Vercel 部署时需要作为环境变量填入。

3.5 本地开发环境配置

在项目根目录创建或编辑 .env.local 文件：

GEMINI_API_KEY=你的LLM_API_Key
VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=你复制的anon_key
🔒
.env.local 已被 .gitignore 排除，不会上传到 GitHub，密钥安全。

本地测试：

npm install
npm run dev

打开浏览器 → 点"管理员入口" → 用管理员邮箱和密码登录。

5 Vercel 部署（网站上线）

  1. 打开 vercel.com，用 GitHub 账号登录
  2. 点 "Add New..." → "Project"

  4. Vercel 自动检测到 Vite，无需修改设置

展开 "Environment Variables"，逐个添加：

  1. 点 "Deploy"
  2. 等待约 30 秒 ~ 1 分钟
  3. 获得公网地址：tsinghua-bid-screener.vercel.app

5.4 自动部署机制（CI/CD）

🔄
修改代码 → git push → Vercel 自动重新构建 → 30 秒内网站更新
无需任何额外配置，这是 Vercel 的核心功能。

5.5 绑定自定义域名（可选）

  1. Vercel 项目 → Settings → Domains

8 项目所有权转移

当需要把项目交给其他人（如新一届负责人）时，需要转移 3 个平台：

#平台转移方式预计时间
1GitHub 仓库转让所有权 或 加 Collaborator2 分钟
2Vercel 项目Transfer Project2 分钟
3Supabase 项目邀请为 Organization Owner2 分钟
⏱️
全部操作约 10 分钟可以完成。

8.1 GitHub 仓库转移

方法 A — 转让所有权（推荐）

  1. 打开 仓库 Settings
  2. 拉到最下面 "Danger Zone"
  3. 点 "Transfer ownership"
  4. 输入接收人的 GitHub 用户名
  5. 确认仓库名称
  6. 对方收到邮件确认后，仓库转到对方名下
  7. 原有链接会自动重定向

方法 B — 加 Collaborator（保留所有权）

  1. 打开 仓库 Access 设置
  2. 点 "Add people"
  3. 输入对方 GitHub 用户名
  4. 权限选 Admin

8.2 Vercel 项目转移

  1. Vercel Dashboard → tsinghua-bid-screener 项目
  2. Settings → General → 拉到底部
  3. 点 "Transfer Project"
  4. 输入对方的 Vercel Team / 用户名
  5. 对方确认接收
✅
环境变量（Supabase URL、Key）会一起转移，无需重新配置。
⚠️
如果同时转让了 GitHub 仓库，需要在 Vercel Settings → Git → Connected Git Repository 中更新关联。

8.3 Supabase 项目转移

方法 A — 直接邀请为 Owner（推荐）

  1. Supabase Dashboard → 左上角 Organization → Settings
  2. "Members" → 邀请对方邮箱为 Owner
  3. 对方接受后，你可以降为 Member 或离开

方法 B — 对方重建

  1. 导出数据：Supabase → Settings → Database → Connection string / Table Editor → Export CSV
  2. 对方创建新项目，运行建表 SQL，导入数据
  3. 更新 Vercel 环境变量 VITE_SUPABASE_URL 和 VITE_SUPABASE_ANON_KEY
  4. 重新部署 Vercel

10 常见问题

Q: 网站打不开 / 加载很慢怎么办？
  • Vercel 在国内可能偶尔较慢，可考虑绑定自定义域名 + CDN
  • 检查 Vercel Dashboard → Deployments 最新部署是否成功
Q: 管理员登录失败怎么办？
  1. 确认邮箱和密码正确
  2. 检查 Supabase → Authentication → Users 中该用户是否为 "Confirmed"
  3. 忘记密码可在 Supabase Users 页面重置
Q: 候选人提交的数据没出现在管理后台？
  1. 检查 Supabase Dashboard → Table Editor → candidates 是否有数据
  2. 如果没有数据，可能是网络问题，数据只存在了 localStorage
  3. 检查 Vercel 环境变量是否正确
Q: 怎么修改网站内容？
  1. 本地用编辑器修改
  2. npm run dev 本地预览
  3. git add . && git commit -m "修改了xxx" && git push
  4. 等 30 秒，网站自动更新
Q: 怎么完全重置系统？
  1. 清空数据：Supabase → SQL Editor → DELETE FROM candidates;
  2. 重置管理员：Authentication → Users → 删除旧用户 → 创建新用户
Q: 环境变量在哪里配置？
环境位置
本地开发项目根目录 .env.local
线上网站Vercel → Settings → Environment Variables
💡
修改线上环境变量后需在 Vercel 重新部署才会生效。
